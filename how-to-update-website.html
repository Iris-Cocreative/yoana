How to update
your website

A step-by-step guide to setting up the Claude app and publishing changes — from your computer or your phone.

You tell Claude what you want changed. Claude writes the code, commits it, and opens a pull request. You review and merge — and your site is live.

1
One-Time Setup — Desktop

Download the Claude desktop app

Start on your computer to connect GitHub. This is a one-time setup — after that you can use your phone for everything.

  1. Download the Claude desktop app at claude.ai/download and sign in.
  2. Open the app and click "New Project" (or select an existing one).
  3. When prompted to connect a repository, click "Connect GitHub."
  4. Authorize Claude to access your GitHub account — this is a one-time step.
  5. Search for and select Iris-Cocreative/yoana — this is your website's repository.
  6. Claude will load your project. You're ready to start chatting.
Already set up? Just open the Claude app and select the Iris-Cocreative/yoana project from your recents. No need to reconnect.
2
Making Changes — Desktop or Phone

Ask Claude to update your site

Open the Claude app on your computer or phone and describe what you want in plain English. You don't need to know anything about code — Claude handles all of that. Here are examples of what you might ask:

Update my bio page with this new text: [paste your text here]
Add a new speaking engagement to the publications page
Change the color of the header buttons to navy blue
Add a new section called "Workshops" to the offerings page
Update my contact email to the new address
Add a new photo to the bio page — here's the file
Tip: You can ask Claude to describe the change first before making it, so you can confirm it sounds right before anything is committed to the site.
Claude — Desktop or Mobile App

Claude: I'll add that to your bio page now. Let me read the current bio first...
✓ Updated bio.html · Committed · Pushed to branch claude/bio-update
✓ Pull request created → github.com/Iris-Cocreative/yoana/pull/12
3
Publishing Changes — From Anywhere

Review & merge the pull request

After Claude finishes, it will create a "pull request" on GitHub — a package of changes for you to review before they go live. You can merge from your computer or directly from your phone using the GitHub mobile app.

  1. Install the GitHub mobile app (free — search "GitHub" in the App Store or Google Play) and sign in. You only need to do this once.
  2. When Claude tells you it's done, open the GitHub app on your phone.
  3. Tap Notifications — you'll see an alert for the new pull request. Tap it to open.
  4. Scroll to the bottom of the pull request and tap "Merge pull request."
  5. Tap "Confirm merge." Done — your site is updated!
Prefer to merge on desktop? Go to github.com/Iris-Cocreative/yoana in a browser. You'll see a yellow banner saying "claude/some-branch had recent pushes" — click "Compare & pull request," then "Create pull request," then the green "Merge pull request" button.
4
After Merging

Your changes go live

Once you merge, your website automatically rebuilds and the changes appear live. This usually takes under a minute. Just refresh your site to see the update.

Something doesn't look right? Just go back to Claude and say "Undo that change" or "Revert the bio page to how it was." Claude will open a new pull request reversing the change — you merge it the same way.
Good to Know

Tips & reminders

Small things that make working with Claude smoother.

💬

Plain language works

You never need to mention code, files, or technical terms. "Change the header" or "add my new headshot" is enough.

↩️

Everything is reversible

Every change is tracked in GitHub. If something goes wrong, Claude can always restore the previous version.

🔀

One session, many changes

You can ask for multiple updates in a single Claude session. Claude keeps track of the conversation and context.

👁️

Preview before publishing

Ask Claude to "describe what the change will look like" before committing. You can approve or redirect before anything is saved.

📱

Works on your phone

Download the Claude app (iOS or Android) and the GitHub app. You can request changes and publish them without ever opening a laptop.

📸

Adding images

You can upload photos directly in the Claude app — on desktop or phone. Tell Claude where you want them placed and it handles the rest.

When You're Ready

Connecting your own domain

Right now your site lives at a GitHub address. When you're ready to move to your own domain (like yoanatchoukleva.com), here's how it works.

1
Buy a Domain

Purchase your domain name

If you don't have one yet, buy a domain from a registrar. Good options are Squarespace Domains, Namecheap, or Google Domains. Search for the name you want and follow their checkout process — it's usually $10–20/year.

Already have a domain? Skip straight to Step 2.
2
Point Your Domain at GitHub

Update your DNS settings

Log in to wherever you bought your domain and find the DNS settings (sometimes called "Manage DNS" or "Advanced DNS"). You need to add four A records pointing to GitHub's servers:

  1. Find the DNS settings panel for your domain.
  2. Delete any existing A records pointing to the root domain (@).
  3. Add four new A records with the host set to @ and these IP addresses, one per record:

    185.199.108.153
    185.199.109.153
    185.199.110.153
    185.199.111.153
  4. Also add a CNAME record with host www pointing to iris-cocreative.github.io
  5. Save your changes. DNS can take up to 24 hours to fully propagate, but usually updates within an hour.
3
Tell GitHub Your Domain

Set the domain in GitHub Pages

  1. Go to github.com/Iris-Cocreative/yoana
  2. Click Settings (top navigation bar of the repository).
  3. Scroll down to the Pages section in the left sidebar.
  4. Under "Custom domain," type your domain name (e.g. yoanatchoukleva.com) and click Save.
  5. Check the box for "Enforce HTTPS" once it becomes available (may take a few minutes). This gives your site a secure padlock in the browser.
After saving, GitHub will verify that the DNS is connected correctly. You'll see a green checkmark when everything is working. If it shows an error, just wait an hour and try again — DNS changes take time to spread.
4
Done

Your site is live at your domain

Once GitHub shows a green checkmark, visit your domain in a browser and you'll see your site. GitHub will automatically redirect iris-cocreative.github.io/yoana to your new domain, so any old links keep working.
